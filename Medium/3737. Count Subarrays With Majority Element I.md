## Problem Statement

Given an integer array `nums` and an integer `target`, return the number of subarrays in which `target` is the majority element.

A majority element in a subarray is an element that appears **strictly more than half** the length of that subarray.

## My Understanding

My first thought was to use the Boyer-Moore Voting Algorithm because the problem mentions a majority element.

However, Boyer-Moore only finds the majority element of the **entire array**.

This problem is different because every subarray can have its own majority element.

For example:

```text
nums = [1,2,2,3]
target = 2
```

The whole array:

```text
[1,2,2,3]
```

does not have a majority element, yet several subarrays have `2` as the majority element.

Therefore, Boyer-Moore is not applicable here.

---

## Key Observation

For a subarray:

```text
nums[i...j]
```

let:

```text
targetCount = occurrences of target
length = j - i + 1
```

Then:

```text
target is majority
iff

targetCount > length / 2
```

So while expanding a subarray, we only need:

* Current length
* Number of occurrences of target

---

## Approach

For every starting index:

```text
i = 0
i = 1
i = 2
...
```

Expand the subarray one element at a time.

Maintain:

```text
targetCount
```

Whenever:

```text
targetCount > length / 2
```

increment the answer.

Since:

```text
n ≤ 1000
```

checking all subarrays is feasible.

## Solution

```cpp
class Solution {
public:
    int countMajoritySubarrays(vector<int>& nums, int target) {

        int n = nums.size();
        int ans = 0;

        for(int i = 0; i < n; i++) {

            int targetCount = 0;

            for(int j = i; j < n; j++) {

                if(nums[j] == target)
                    targetCount++;

                int len = j - i + 1;

                if(targetCount > len / 2)
                    ans++;
            }
        }

        return ans;
    }
};
```

| Metric           | Complexity |
| ---------------- | ---------- |
| Time Complexity  | O(n²)      |
| Space Complexity | O(1)       |

The condition:


directly determines whether `target` is the majority element.

No frequency map is required because we only care about the count of one specific value.

### Majority Condition

```text
targetCount > length / 2
```

not:

```text
targetCount >= length / 2
```

The problem requires:

```text
strictly more than half
```

### Why Boyer-Moore Doesn't Work

Boyer-Moore finds the majority element of a single array.

This problem requires checking:

```text
O(n²) different subarrays
```

each with its own majority condition.
