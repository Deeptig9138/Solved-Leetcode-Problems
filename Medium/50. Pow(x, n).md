## Problem Statement

Implement `pow(x, n)`, which calculates `x` raised to the power `n` (`xⁿ`).

### Constraints

* `-100.0 < x < 100.0`
* `-2³¹ <= n <= 2³¹ - 1`
* `n` is an integer.
* Either `x` is not zero or `n > 0`.

## My Understanding

My first thought was to use the built-in function:

```cpp
return pow(x, n);
```

But the purpose of this problem is to implement the power function ourselves.

The next straightforward idea would be:

```cpp
ans = 1;

for(int i = 0; i < n; i++)
    ans *= x;
```

This works, but after looking at the constraints, I realized that:

```text
n can be as large as 2,147,483,647
```

A linear solution would require billions of multiplications, making it far too slow.

So the challenge is not calculating powers—it is calculating them efficiently.

---

## Key Observation

Instead of multiplying `x` repeatedly, we can use previously computed powers.

For example:

```text
x¹⁰
```

can be rewritten as:

```text
x¹⁰ = (x⁵)²
```

Similarly:

```text
x⁸ = (x⁴)²
x⁴ = (x²)²
x² = (x¹)²
```

The exponent keeps getting divided by 2.

This suggests a logarithmic solution instead of a linear one.

---

## Approach

### Binary Exponentiation (Fast Power)

For an exponent `n`:

#### If `n` is even

```text
xⁿ = (xⁿ⁄²)²
```

#### If `n` is odd

```text
xⁿ = x × xⁿ⁻¹
```

Instead of processing every power individually, we repeatedly:

1. Check if the current exponent is odd.
2. Multiply the answer when needed.
3. Square the base.
4. Divide the exponent by 2.

This reduces the number of operations dramatically.

### Handling Negative Powers

For example:

```text
2⁻²
```

means:

```text
1 / (2²)
```

So:

```cpp
x = 1/x;
n = -n;
```

However, there is a special edge case:

```text
n = -2³¹
```

Negating this value overflows a 32-bit integer.

To avoid this, we first store `n` in a `long long`.

```cpp
long long N = n;
```

## Dry Run

### Input

```text
x = 2
n = 10
```

### Initial State

```text
ans = 1
x = 2
N = 10
```

| N  | Odd? | ans  | x     | Action                 |
| -- | ---- | ---- | ----- | ---------------------- |
| 10 | No   | 1    | 4     | Square x               |
| 5  | Yes  | 4    | 16    | Multiply ans, square x |
| 2  | No   | 4    | 256   | Square x               |
| 1  | Yes  | 1024 | 65536 | Multiply ans           |

### Final Answer

```text
1024
```

---

## Solution

```cpp
class Solution {
public:
    double myPow(double x, int n) {

        long long N = n;

        if (N < 0) {
            x = 1.0 / x;
            N = -N;
        }

        double ans = 1;

        while (N > 0) {

            if (N % 2 == 1)
                ans *= x;

            x *= x;
            N /= 2;
        }

        return ans;
    }
};
```

## Complexity Analysis

| Metric           | Complexity |
| ---------------- | ---------- |
| Time Complexity  | O(log n)   |
| Space Complexity | O(1)       |

## Patterns Learned

* Binary Exponentiation reduces repeated multiplication dramatically.
* Repeated squaring is a powerful optimization technique.
* Large constraints often indicate that an O(n) solution is insufficient.
* When a problem involves powers, think about dividing the exponent rather than multiplying repeatedly.

## Revision Notes

* Brute force multiplication takes O(n).
* Constraints immediately suggest a faster approach is needed.
* Binary Exponentiation works by repeatedly halving the exponent.
* Always handle negative powers separately.
* Remember the overflow case:

```text
n = -2³¹
```

Use:

```cpp
long long N = n;
```

before negating.

### Interview Takeaway

When an interviewer asks for:

```text
xⁿ
```

and `n` can be very large:

```text
Don't think repeated multiplication.
Think Binary Exponentiation.
```

This is one of the most common Divide-and-Conquer optimization patterns in coding interviews.
