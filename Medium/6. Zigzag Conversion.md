## Problem Statement

The string `"PAYPALISHIRING"` is written in a zigzag pattern on a given number of rows.

For example, with `numRows = 3`:

```text
P   A   H   N
A P L S I I G
Y   I   R
```

Reading line by line produces:

```text
PAHNAPLSIIGYIR
```

Given a string `s` and an integer `numRows`, return the zigzag-converted string.

## My Understanding

My initial thought was to group characters based on the number of rows.

For example, if there are 3 rows, I considered taking:

```text
first character and every 3rd character after it
second character and every 3rd character after it
third character and every 3rd character after it
```

and then combining the results.

At first glance this seems reasonable because the output is eventually read row by row.

However, after testing it on:

```text
PAYPALISHIRING
```

I noticed the positions in each row are not separated by a constant gap.

The zigzag pattern doesn't simply move downward.

It moves:

```text
down ↓
then
up diagonally ↗
then
down ↓
then
up diagonally ↗
```

This means characters "bounce" between rows, making direct slicing difficult.

---

## Key Observation

The easiest way to think about the problem is:

```text
Don't calculate positions.
Simulate the zigzag movement.
```

Create one string for each row and place each character into its current row.

Track:

```text
currentRow
direction
```

When we reach:

```text
row 0
```

we start moving down.

When we reach:

```text
row numRows - 1
```

we start moving up.

### Why the "Every numRows Character" Idea Fails

Consider:

```text
PAYPALISHIRING
```

Indices:

```text
0:P
1:A
2:Y
3:P
4:A
5:L
6:I
7:S
8:H
9:I
10:R
11:I
12:N
13:G
```

If we take every third character starting from index 0:

```text
0, 3, 6, 9, 12
```

we get:

```text
PPIIN
```

But the actual first row is:

```text
PAHN
```

The spacing is not constant because the zigzag changes direction.

## Approach

### Simulation Using Row Strings

1. Create `numRows` strings.
2. Start at row 0.
3. Traverse the input string character by character.
4. Append the current character to the current row.
5. Move downward until the last row.
6. Reverse direction and move upward.
7. Repeat until all characters are processed.
8. Concatenate all row strings.

## Solution

```cpp
class Solution {
public:
    string convert(string s, int numRows) {

        if(numRows == 1 || numRows >= s.size())
            return s;

        vector<string> rows(numRows);

        int currentRow = 0;
        bool goingDown = true;

        for(char ch : s) {

            rows[currentRow] += ch;

            if(currentRow == 0)
                goingDown = true;
            else if(currentRow == numRows - 1)
                goingDown = false;

            if(goingDown)
                currentRow++;
            else
                currentRow--;
        }

        string result = "";

        for(string row : rows)
            result += row;

        return result;
    }
};
```

## Complexity Analysis

| Metric           | Complexity |
| ---------------- | ---------- |
| Time Complexity  | O(n)       |
| Space Complexity | O(n)       |

where `n` is the length of the string.

## Patterns Learned

* Simulation is sometimes simpler than deriving a mathematical formula.
* When a pattern changes direction dynamically, direct indexing becomes difficult.
* Maintaining state (`currentRow`, `direction`) often leads to cleaner solutions.
* Building intermediate structures and combining them later can simplify implementation.

## Revision Notes

* First intuition: group characters row-wise using fixed jumps.
* Issue: zigzag rows do not have constant spacing.
* Better approach: simulate the movement.
* Maintain:

```text
currentRow
goingDown
```

* Change direction at:

```text
row 0
row numRows - 1
```

### Interview Takeaway

When a problem describes movement like:

```text
down
up
down
up
```

don't force a mathematical indexing solution immediately.

Often the cleanest solution is:

```text
Simulate the process exactly as described.
```

This produces a simple O(n) solution that is easy to explain and implement.
