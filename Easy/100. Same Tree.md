## Problem Statement

Given the roots of two binary trees `p` and `q`, determine whether they are identical.

Two binary trees are considered the same if:

* They have the same structure.
* Corresponding nodes contain the same value.

Return `true` if both trees are identical; otherwise, return `false`.

## My Understanding

My first thought was simple:

> Compare both the structure of the trees and the value stored in each corresponding node.

If at any point:

* one tree has a node while the other doesn't, or
* both nodes exist but their values are different,

then the trees cannot be identical.

Since every node has two children, this naturally suggests recursively comparing both the left and right subtrees.

---

## Key Observation

For every pair of corresponding nodes `(p, q)`, only four cases are possible:

### Case 1

Both nodes are `NULL`.

```text
NULL    NULL
```

These are identical.

Return:

```text
true
```

### Case 2

One node is `NULL` while the other isn't.

```text
NULL    5
```

Structures differ.

Return:

```text
false
```

### Case 3

Both nodes exist but values differ.

```text
4    7
```

Values differ.

Return:

```text
false
```

### Case 4

Both nodes exist and values are equal.

```text
4    4
```

Continue checking:

* Left subtree
* Right subtree

Both must also be identical.

---

## Approach

Use Depth-First Search (DFS) with recursion.

For each pair of nodes:

1. If both nodes are `NULL`, return `true`.
2. If only one node is `NULL`, return `false`.
3. If their values differ, return `false`.
4. Recursively compare:

   * Left subtree
   * Right subtree
5. Return the logical AND of both recursive calls.

The trees are identical only if every corresponding node satisfies these conditions.

## Solution

```cpp
class Solution {
public:
    bool isSameTree(TreeNode* p, TreeNode* q) {

        if(p == NULL && q == NULL)
            return true;

        if(p == NULL || q == NULL)
            return false;

        if(p->val != q->val)
            return false;

        return isSameTree(p->left, q->left) &&
               isSameTree(p->right, q->right);
    }
};
```

## Complexity Analysis

Let `n` be the number of nodes in the trees.

| Metric           | Complexity |
| ---------------- | ---------- |
| Time Complexity  | O(n)       |
| Space Complexity | O(h)       |

where `h` is the height of the tree.

* Balanced tree: **O(log n)**
* Worst case (skewed tree): **O(n)**

The extra space comes from the recursive call stack.

---

## Key Observation

This problem is not about traversing one tree independently.

Instead, we traverse **both trees simultaneously**, comparing corresponding nodes at every recursive call.

If any mismatch in structure or value is found, we can immediately return `false`.

---

### Interview Takeaway

Whenever a problem asks whether two binary trees are identical, think:

```text
Compare corresponding nodes recursively.
```

Don't compare one tree first and then the other.

Traverse both trees simultaneously.
