## Problem Statement

There is a biker going on a road trip.

The biker starts at point `0` with an altitude of:

```text
0
```

You are given an integer array `gain` where:

```text
gain[i]
```

represents the net gain (or loss) in altitude between point `i` and point `i + 1`.

Return the highest altitude reached during the trip.

## My Understanding

My first thought was to build the complete altitude array.

Since the biker starts at:

```text
Altitude = 0
```

I can keep adding the gains one by one:

Example:

```text
gain = [5,6,-7,8]
```

Starting from:

```text
0
```

Altitudes become:

```text
[0,
 0+5,
 0+5+6,
 0+5+6-7,
 0+5+6-7+8]
```

Which gives:

```text
[0, 5, 11, 4, 12]
```

Then simply find the largest value in the altitude array.

This works perfectly.

After thinking further, I realized that storing the entire altitude array is unnecessary because we only need:

* Current altitude
* Highest altitude seen so far

---

## Key Observation

Every altitude is simply a running sum of the `gain` array.

If:

```text
currentAltitude += gain[i]
```

then:

```text
highestAltitude = max(highestAltitude, currentAltitude)
```

So we can compute the answer while traversing the array once.

---

## Approach

### Running Prefix Sum

1. Start with:

   * `currentAltitude = 0`
   * `maxAltitude = 0`
2. Traverse the `gain` array.
3. Add the current gain to the current altitude.
4. Update the maximum altitude if needed.
5. Return the maximum altitude.

| -7   | -6               | 1                |

```text
[0, -5, -4, 1, 1, -6]
```

```text
1
```

```text
1
```

## Solution

```cpp
class Solution {
public:
    int largestAltitude(vector<int>& gain) {

        int currentAltitude = 0;
        int maxAltitude = 0;

        for(int g : gain) {
            currentAltitude += g;
            maxAltitude = max(maxAltitude, currentAltitude);
        }

        return maxAltitude;
    }
};
```

## Complexity Analysis

| Metric           | Complexity |
| ---------------- | ---------- |
| Time Complexity  | O(n)       |
| Space Complexity | O(1)       |

where `n` is the length of the `gain` array.

---

## Key Observation

The altitude at any point is simply the prefix sum of all gains up to that point.

Instead of storing every altitude:

```text
[0, -5, -4, 1, 1, -6]
```

we only keep:

```text
current altitude
highest altitude
```

which reduces space usage.

### Interview Takeaway

Whenever a problem asks for:

```text
running totals
cumulative values
sum up to current position
```

think:

```text
Prefix Sum
```

And if only the final answer is needed, ask yourself:

```text
Do I really need to store the entire prefix array?
```

Often, a running sum is enough.
