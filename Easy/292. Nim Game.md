# 292. Nim Game

**Difficulty :** Easy

## Topics / Patterns

* Math
* Game Theory

## Problem Statement

You are playing the following Nim Game with your friend:

* There is a heap of stones on the table.
* You and your friend take turns removing stones.
* You go first.
* On each turn, a player may remove 1, 2, or 3 stones.
* The player who removes the last stone wins.

Return `true` if you can win assuming both players play optimally, otherwise return `false`.

### Constraints

* `1 <= n <= 2³¹ - 1`

---

**Input**

```text
n = 4
```

**Output**

```text
false
```

**Explanation**

No matter what move you make:

```text

4 → 2
4 → 1
```

your opponent can take the remaining stones and win.


**Output**


### Example 3


**Output**

```text
true
```

## My Understanding

My initial thought was:

Since I go first and can remove 1, 2, or 3 stones, then for:

```text
n = 1, 2, 3
```

I automatically win because I can take all the stones.

But for larger values, I realized that my move alone isn't enough to determine the result.

The important part of the problem is:

```text
Both players play optimally.
```

That means I need to think about what position I leave for my opponent after my move.

Instead of focusing on what I can take, I need to identify which positions are winning and losing.

---

## Key Observation

Let's build a pattern.

| Stones | Result |
| ------ | ------ |
| 1      | Win    |
| 2      | Win    |
| 3      | Win    |
| 4      | Lose   |

### Why is 4 losing?

Possible moves:

```text
4 → 3
4 → 2
4 → 1
```

All of these positions are winning for the next player.

Therefore:

```text
4 = Losing Position
```

Continue the pattern:

### n = 5

Take 1 stone:

```text
5 → 4
```

Opponent gets a losing position.

So:

```text
5 = Winning Position
```

### n = 6

Take 2 stones:

```text
6 → 4
```

Opponent gets a losing position.

So:

```text
6 = Winning Position
```

### n = 7

Take 3 stones:

```text
7 → 4
```

Opponent gets a losing position.

So:

```text
7 = Winning Position
```

### n = 8

Possible moves:

```text
8 → 7
8 → 6
8 → 5
```

But:

```text
5, 6, and 7
```

are all winning positions.

So:

```text
8 = Losing Position
```

## Pattern Discovery

| n  | Result |
| -- | ------ |
| 1  | W      |
| 2  | W      |
| 3  | W      |
| 4  | L      |
| 5  | W      |
| 6  | W      |
| 7  | W      |
| 8  | L      |
| 9  | W      |
| 10 | W      |
| 11 | W      |
| 12 | L      |

A clear pattern appears:

```text
Every multiple of 4 is a losing position.
```

### Why Does This Work?

Suppose you start with:

```text
4k stones
```

No matter what you remove:

```text
1, 2, or 3
```

you leave:

```text
4k - 1
4k - 2
4k - 3
```

Your opponent can always remove:

```text
3, 2, or 1
```

respectively and bring the pile back to:

```text
4(k - 1)
```

another multiple of 4.

The opponent can keep repeating this strategy until they win.

Therefore:

```text
Multiples of 4 are losing positions.
```

Every other number has at least one move that reaches a multiple of 4.

---

## Approach

1. Check if `n` is divisible by 4.
2. If yes, return `false`.
3. Otherwise, return `true`.

No simulation is needed once the pattern is discovered.

---

## Dry Run

### Input

```text
n = 8
```

Since:

```text
8 % 4 == 0
```

8 is a losing position.

### Answer

```text
false
```

### Input

```text
n = 10
```

Since:

```text
10 % 4 = 2
```

Take 2 stones:

```text
10 → 8
```

Opponent receives a losing position.

### Answer

```text
true
```

---

## Solution

```cpp
class Solution {
public:
    bool canWinNim(int n) {
        return n % 4 != 0;
    }
};
```

## Revision Notes

* Start by analyzing small cases.
* Mark positions as Winning (W) or Losing (L).
* A position is winning if you can move to a losing position.
* A position is losing if all possible moves lead to winning positions.
* Pattern discovered:

```text
1 W
2 W
3 W
4 L
5 W
6 W
7 W
8 L
...
```

### Interview Takeaway

When a game problem appears:

```text
Don't immediately think DP.
First build small cases and look for a pattern.
```

For Nim Game specifically:

```text
n % 4 == 0 → Lose
n % 4 != 0 → Win
```

The entire solution reduces to a single mathematical observation.
