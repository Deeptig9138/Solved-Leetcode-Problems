## Problem Statement

A happy number is defined by the following process:

1. Start with any positive integer.
2. Replace the number with the sum of the squares of its digits.
3. Repeat the process until:

   * The number becomes `1`, or
   * The process enters a cycle that never includes `1`.

Return:

```text
true
```

if the number is happy, otherwise return:

```text
false
```

---

### Example 1

**Input**


**Output**

```text

```

**Explanation**

```text

8² + 2² = 68
6² + 8² = 100
1² + 0² + 0² = 1
```

Since the process reaches `1`, the number is happy.

## My Understanding

My first observation was that we repeatedly:

1. Extract each digit.
2. Square it.
3. Add all squared digits together.
4. Repeat the process.

For example:

```text
19
```

becomes:

```text
1² + 9² = 82
```

and then:

```text
8² + 2² = 68
```

and so on.

Initially, it might seem like we can stop when the result becomes a single digit, but that is incorrect.

Example:

```text
7
```

is already a single digit, but:

```text
7
→ 49
→ 97
→ 130
→ 10
→ 1
```

So a single digit does not necessarily mean we are done.

The real challenge is determining whether the process eventually reaches `1` or falls into an infinite cycle.

---

## Key Observation

Every generated number will eventually do one of two things:

### Case 1: Reach 1

Example:

```text
19
→ 82
→ 68
→ 100
→ 1
```

Answer:

```text

```

### Case 2: Enter a Cycle

Example:

```text
2
→ 4
→ 16
→ 37
→ 58
→ 89
→ 145
→ 42
→ 20
→ 4
```

Notice:

```text

```

Once a number repeats, the entire sequence will repeat forever.

Answer:

```text
false
```

## Approach

### Hash Set + Cycle Detection

1. Create a hash set called `seen`.
2. Generate the next number using the sum of squared digits.
3. Before processing:

   * If the number becomes `1`, return `true`.
   * If the number already exists in `seen`, return `false`.
4. Otherwise store the number in the set.
5. Continue the process.

## Solution

```cpp
class Solution {
public:

    int getNext(int n) {

        int sum = 0;

        while(n > 0) {
            int digit = n % 10;
            sum += digit * digit;
            n /= 10;
        }

        return sum;
    }

    bool isHappy(int n) {

        unordered_set<int> seen;

        while(n != 1 && !seen.count(n)) {

            seen.insert(n);
            n = getNext(n);
        }

        return n == 1;
    }
};
```

## Complexity Analysis

Let:

```text
k = number of generated states
```

before reaching `1` or entering a cycle.

| Metric           | Complexity |
| ---------------- | ---------- |
| Time Complexity  | O(k)       |
| Space Complexity | O(k)       |

In practice, `k` remains very small.

## Alternative Solution (Floyd's Cycle Detection)

Since this problem is essentially:

```text
Generate next state
→ Generate next state
→ Generate next state
```

it behaves similarly to a linked list.

Therefore we can use:

```text
Slow Pointer
Fast Pointer
```

just like the Linked List Cycle problem.

### Idea

* Slow moves one step.
* Fast moves two steps.
* If they meet before reaching `1`, a cycle exists.
* If either reaches `1`, the number is happy.

This reduces space complexity to:

```text
O(1)
```

