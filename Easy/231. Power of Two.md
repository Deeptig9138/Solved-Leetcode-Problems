# 231. Power of Two

**Difficulty :** Easy

## Topics / Patterns

* Math
* Bit Manipulation

## Problem Statement

Given an integer `n`, return `true` if it is a power of two. Otherwise, return `false`.

An integer `n` is a power of two if there exists an integer `x` such that:

```text
n = 2^x
```

---

## Examples

### Example 1

**Input**

```text
n = 1
```

**Output**

```text
true
```

**Explanation**

```text
2^0 = 1
```

### Example 2

**Input**

```text
n = 16
```

**Output**

```text
true
```

**Explanation**

```text
2^4 = 16
```

### Example 3

**Input**

```text
n = 3
```

**Output**

```text
false
```

---

## My Understanding

My first observation was:

```text
1 = 2^0
2 = 2^1
4 = 2^2
8 = 2^3
16 = 2^4
...
```

I also noticed that except for:

```text
1
```

every power of two is even.

However, that observation alone is not enough because:

```text
12
```

is also even, but:

```text
12 ≠ 2^x
```

So simply checking whether a number is odd or even cannot solve the problem.

The next idea was to repeatedly divide by 2 and see if the number eventually becomes 1.

---

## Key Observation

A power of two can always be reduced to:

```text
1
```

by repeatedly dividing by 2.

Example:

```text
16
→ 8
→ 4
→ 2
→ 1
```

But:

```text
12
→ 6
→ 3
```

At this point it is no longer divisible by 2.

Therefore:

```text
12 is not a power of two
```

---

## Approach 1: Repeated Division

### Steps

1. If `n <= 0`, return `false`.
2. Keep dividing by 2 while the number is divisible by 2.
3. After the loop:

   * If `n == 1`, return `true`.
   * Otherwise return `false`.

---

## Dry Run

### Input

```text
n = 16
```

Process:

```text
16 → 8 → 4 → 2 → 1
```

Result:

```text
true
```

### Input

```text
n = 12
```

Process:

```text
12 → 6 → 3
```

Cannot divide further.

Result:

```text
false
```

---

## Solution (Division Method)

```cpp
class Solution {
public:
    bool isPowerOfTwo(int n) {

        if(n <= 0)
            return false;

        while(n % 2 == 0)
            n /= 2;

        return n == 1;
    }
};
```

## Complexity Analysis

| Metric           | Complexity |
| ---------------- | ---------- |
| Time Complexity  | O(log n)   |
| Space Complexity | O(1)       |

## Follow-Up: No Loops / Recursion

The follow-up hints toward a bit manipulation solution.

## Binary Representation Observation

Look at powers of two in binary:

```text
1  = 0001
2  = 0010
4  = 0100
8  = 1000
16 = 10000
```

Notice something interesting:

```text
Every power of two has exactly ONE set bit.
```

### What Happens When We Subtract 1?

Example:

```text
8     = 1000
8 - 1 = 0111
```

Perform AND operation:

```text
1000
0111
----
0000
```

Result:

```text
0
```

This happens for every power of two.

## Key Bit Manipulation Property

```text
n & (n - 1) == 0
```

is true only when `n` contains exactly one set bit.

Since powers of two have exactly one set bit:

```text
n > 0 && (n & (n - 1)) == 0
```

correctly identifies powers of two.

---

## Dry Run (Bit Method)

### Input

```text
10000
```

```text
true
```

```text

```

Not zero.

Result:

```text
false
```

---

## Optimized Solution (Bit Manipulation)

```cpp
class Solution {
public:
    bool isPowerOfTwo(int n) {

        return n > 0 && (n & (n - 1)) == 0;
    }
};
```

| Metric           | Complexity |
| ---------------- | ---------- |
| Time Complexity  | O(1)       |
| Space Complexity | O(1)       |

## Revision Notes

### Mathematical Observation

```text
Power of Two
→ Keep dividing by 2
→ Must eventually reach 1
```

### Binary Observation

```text
1  = 0001
2  = 0010
4  = 0100
8  = 1000
```

Exactly one bit is set.

### Important Formula

```text
n & (n - 1) == 0
```

works because subtracting 1 flips the only set bit and all bits after it.

### Edge Case

```text
n <= 0
```

must always return:

```text
false
```

because powers of two are positive.

### Interview Takeaway

Start with the division approach because it is intuitive.

Then mention the optimized observation:

```text
A power of two contains exactly one set bit.
```

which leads to:

```cpp
n > 0 && (n & (n - 1)) == 0
```

This is one of the most common bit manipulation patterns asked in interviews.
