## Problem Statement

Given a string `s` consisting of words and spaces, return the length of the **last word**.

A word is defined as a maximal substring consisting of non-space characters only.

## My Understanding

My first thought was to traverse the string until I reached the last word and then simply calculate its length.

However, the input is a single string, not separate words, so I first need to identify where the last word begins and ends.

Instead of splitting the string into multiple words (which requires extra space), I can traverse from the end of the string. By skipping any trailing spaces and then counting characters until I reach another space, I obtain the length of the last word directly.

---

## Key Observation

The only information we need is the **length of the last word**.

We do **not** need:

* All words
* A vector of strings
* Any substring operations

Scanning backwards is enough.

---

## Approach

1. Start from the last character of the string.
2. Skip all trailing spaces.
3. Count characters until a space or the beginning of the string is reached.
4. Return the count.

This avoids creating additional strings and uses constant extra space.

## Solution

```cpp
class Solution {
public:
    int lengthOfLastWord(string s) {

        int i = s.length() - 1;

        while(i >= 0 && s[i] == ' ')
            i--;

        int len = 0;

        while(i >= 0 && s[i] != ' ') {
            len++;
            i--;
        }

        return len;
    }
};
```

## Complexity Analysis

| Metric           | Complexity |
| ---------------- | ---------- |
| Time Complexity  | O(n)       |
| Space Complexity | O(1)       |

Although we traverse from the end, in the worst case every character may be visited once.

---

## Key Observation

The last word is always located by:

1. Ignoring trailing spaces.
2. Counting characters until the next space.

No additional string manipulation is required.

## Revision Notes

### Algorithm

```text
Start from the end
↓
Skip trailing spaces
↓
Count characters
↓
Stop at space
↓
Return count
```

### Why Reverse Traversal?

If we scan from the beginning, we'd have to keep track of every word encountered.

Scanning from the end allows us to find the answer immediately without storing anything.

### Interview Takeaway

Whenever a problem asks about the **last** word, character, or token in a string, consider traversing **backwards**. Reverse traversal often leads to a simpler and more space-efficient solution.
