## Problem Statement

Given an array `nums` of size `n`, return the majority element.

The majority element is the element that appears more than:

```text
⌊n / 2⌋
```

times.

You may assume that the majority element always exists in the array.

---

**Input**

```text
nums = [3,2,3]
```

**Output**

```text
3
```

### Example 2

**Input**

```text
nums = [2,2,1,1,1,2,2]
```

**Output**

```text
2
```

## My Understanding

My first idea was to count the frequency of every element using a hash map.

For example:

```text
[2,2,1,1,1,2,2]
```

Frequency table:

```text
2 → 4
1 → 3
```

Array size:

```text
n = 7
```

Majority condition:

```text
frequency > n/2
frequency > 3
```

Since:

```text
2 appears 4 times
```

it is the majority element.

This approach is simple and works efficiently.

---

## Approach 1: Hash Map Counting

### Idea

1. Traverse the array.
2. Store the frequency of every element in a hash map.
3. Traverse the frequency table.
4. Return the element whose frequency exceeds:

```text
n / 2
```

## Solution (Hash Map)

```cpp
class Solution {
public:
    int majorityElement(vector<int>& nums) {

        unordered_map<int, int> freq;

        for(int num : nums) {
            freq[num]++;
        }

        int n = nums.size();

        for(auto it : freq) {
            if(it.second > n / 2)
                return it.first;
        }

        return -1;
    }
};
```

## Follow-Up: O(1) Space Solution

The follow-up asks:

```text
Can you solve it in O(n) time and O(1) space?
```

A hash map uses extra memory.

So we need a different approach.

---

## Key Observation

Since the majority element appears more than half the time:

```text
majority frequency > n/2
```

it can never be completely cancelled by all other elements.

Imagine:

```text
Majority element = +1
Any different element = -1
```

Whenever we encounter:

```text
same element
```

we increase a counter.

Whenever we encounter:

```text
different element
```

we decrease the counter.

All non-majority elements eventually cancel out, leaving the majority element behind.

This is the intuition behind the Boyer-Moore Voting Algorithm.

---

## Approach 2: Boyer-Moore Voting Algorithm

Maintain:

```text
candidate
count
```

Rules:

### If count becomes 0

Choose the current element as the new candidate.

### If current element equals candidate

```text
count++
```

### Otherwise

```text
count--
```

At the end, the candidate will be the majority element.

| 1       | 1         | 1     |
| 2       | 1         | 0     |
| 2       | 2         | 1     |

2

## Optimized Solution (Boyer-Moore)

```cpp
class Solution {
public:
    int majorityElement(vector<int>& nums) {

        int candidate = 0;
        int count = 0;

        for(int num : nums) {

            if(count == 0)
                candidate = num;

            if(num == candidate)
                count++;
            else
                count--;
        }

        return candidate;
    }
};
```

## Complexity Analysis

| Metric           | Complexity |
| ---------------- | ---------- |
| Time Complexity  | O(n)       |
| Space Complexity | O(1)       |

---

## Why Boyer-Moore Works

The majority element appears more than half the time.

Therefore:

```text
Majority Count >
All Other Counts Combined
```

Even after repeatedly cancelling one majority element with one non-majority element:

```text
Majority elements still remain.
```

So the final surviving candidate must be the majority element.
