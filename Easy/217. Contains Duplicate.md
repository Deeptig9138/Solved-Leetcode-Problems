## Problem Statement

Given an integer array `nums`, return `true` if any value appears at least twice in the array, and return `false` if every element is distinct.

### Constraints

* `1 <= nums.length <= 10^5`
* `-10^9 <= nums[i] <= 10^9`

---

## Examples

### Example 1

**Input**

```text
nums = [1,2,3,1]
```

**Output**

```text
true
```

**Explanation**

The element `1` appears more than once in the array.

### Example 2

```text
true
```

## My Understanding

My first thought was straightforward:

For every element, scan the remaining array and check if the same value appears again.

Something like:

```text
Pick a number → Search the rest of the array for the same number
```

This approach would work correctly, but after considering the constraint `n = 10^5`, it became clear that repeatedly scanning the array would be too slow.

So instead of searching every time, I needed a way to remember which numbers had already been seen.

That naturally led to using a Hash Table (Hash Set).

---

## Key Observation

The moment we encounter a number that has already appeared before, we can immediately return `true`.

A Hash Set allows us to check whether an element has been seen before in approximately O(1) time.

---

## Approach

### Brute Force Idea

For every element:

1. Compare it with every other element after it.
2. If a match is found, return `true`.
3. Otherwise continue.

This works but takes O(n²) time.

---

### Optimized Hash Set Approach

1. Create an empty hash set.
2. Traverse the array.
3. For each number:

   * Check if it already exists in the set.
   * If it does, return `true`.
   * Otherwise insert it into the set.
4. If traversal finishes without finding duplicates, return `false`.

This reduces the lookup time significantly.

---

## Dry Run

### Input

```text
nums = [1,2,3,1]
```

### Initial State

```text
set = {}
```

| Element | Set Before | Action                       |
| ------- | ---------- | ---------------------------- |
| 1       | {}         | Insert 1                     |
| 2       | {1}        | Insert 2                     |
| 3       | {1,2}      | Insert 3                     |
| 1       | {1,2,3}    | Already exists → Return true |

### Final Answer

```text
true
```

---

## Solution

```cpp
class Solution {
public:
    bool containsDuplicate(vector<int>& nums) {

        unordered_set<int> seen;

        for(int num : nums) {

            if(seen.find(num) != seen.end())
                return true;

            seen.insert(num);
        }

        return false;
    }
};
```

## Complexity Analysis

### Optimized Hash Set Solution

| Metric           | Complexity |
| ---------------- | ---------- |
| Time Complexity  | O(n)       |
| Space Complexity | O(n)       |

### Brute Force Solution

| Metric           | Complexity |
| ---------------- | ---------- |
| Time Complexity  | O(n²)      |
| Space Complexity | O(1)       |

## Revision Notes

* First approach was nested iteration (O(n²)).
* Constraints hint that a more efficient solution is needed.
* Hash Set provides O(1) average lookup time.
* As soon as a duplicate is found, return immediately.
* Common interview pattern:

```text
Need fast existence checking?
→ Think Hash Set / Hash Map
```

* Alternative solution:

```cpp
unordered_set<int> s(nums.begin(), nums.end());
return s.size() != nums.size();
```

This works because a set automatically removes duplicate values.
