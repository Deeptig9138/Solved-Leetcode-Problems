## Problem Statement

Given an array of integers `nums`, return the **pivot index** of this array.

The pivot index is the index where the sum of all elements strictly to the left is equal to the sum of all elements strictly to the right.

If the index is on the left edge, left sum is 0. Same for the right edge.

Return the **leftmost pivot index**. If no such index exists, return `-1`.

## My Understanding

My initial instinct was to think about searching or trying to “find a center” in the array — maybe even starting from the middle or using some kind of scanning technique.

But quickly it became clear that:

* There is no ordering property in the array
* The pivot is not guaranteed to be near the center
* Values matter only in terms of sums, not positions or frequency

So this is not a searching problem — it is a **balance of sums problem**.

---

## Key Observation

For an index `i`:

```text id="l7m2ab"
left_sum == right_sum
```

Instead of recomputing left and right sums every time, we use the total sum of the array.

Let:

```text id="v4p9cd"
total = sum(nums)
left = sum of elements before index i
```

Then:

```text id="n8q1zz"
right = total - left - nums[i]
```

So the condition becomes:

```text id="t5r3lm"
left == total - left - nums[i]
```

This eliminates the need for nested loops.

---

## Approach

### Optimal Prefix Sum Approach

1. Compute total sum of the array.

2. Initialize `leftSum = 0`.

3. Traverse the array from left to right.

4. For each index `i`, check:

   * If `leftSum == total - leftSum - nums[i]`, return `i`.

5. Otherwise, add `nums[i]` to `leftSum`.

6. If no pivot is found, return `-1`.

## Solution

```cpp id="p3q7mn"
class Solution {
public:
    int pivotIndex(vector<int>& nums) {

        int total = 0;
        for (int x : nums) total += x;

        int leftSum = 0;

        for (int i = 0; i < nums.size(); i++) {

            if (leftSum == total - leftSum - nums[i])
                return i;

            leftSum += nums[i];
        }

        return -1;
    }
};
```

## Patterns Learned

* Prefix sum is useful when problems involve “left vs right” comparisons.
* Total sum can help avoid recomputing suffix sums.
* Balance equations often simplify to a single-pass solution.
* If a problem involves equality between two sides of an array, think prefix sum first.
