# [125. Valid Palindrome](https://leetcode.com/problems/valid-palindrome/description/)

**Difficulty :** Easy

## Topics / Patterns

## Problem Statement

A phrase is considered a palindrome if, after:

1. Converting all uppercase letters to lowercase.
2. Removing all non-alphanumeric characters.

it reads the same forward and backward.

Given a string `s`, return `true` if it is a palindrome, otherwise return `false`.

---

## Examples

### Example 1

**Input**

```text
s = "A man, a plan, a canal: Panama"
```

**Output**

```text
true
```

**Explanation**

After removing non-alphanumeric characters and converting to lowercase:

```text
amanaplanacanalpanama
```

This reads the same forward and backward.

### Example 2

**Input**

```text
s = "race a car"
```

**Output**

```text
false
```

**Explanation**

Cleaned string:

```text
raceacar
```

This is not a palindrome.

### Example 3

**Input**

```text
s = " "
```

**Output**

```text
true
```

**Explanation**

After removing non-alphanumeric characters:

```text
""
```

An empty string is considered a palindrome.

## My Understanding

My first thought was to create a new string by:

1. Converting every character to lowercase.
2. Removing spaces and special characters.

Then I could simply check whether the resulting string is a palindrome using two pointers.

While this works, it requires extra space to store the cleaned string.

A better approach is to use two pointers directly on the original string and skip invalid characters during comparison.

---

## Key Observation

The problem states:

```text
Ignore:
- Uppercase vs lowercase
- Special characters
- Spaces
```

Therefore:

```text
'A' == 'a'
```

and

```text
',', ':', ' '
```

should not participate in comparisons.

This naturally suggests a two-pointer approach.

---

## Approach

Use two pointers:

```text
left  = 0
right = s.length() - 1
```

While `left < right`:

1. Skip non-alphanumeric characters from the left.
2. Skip non-alphanumeric characters from the right.
3. Convert both characters to lowercase.
4. Compare them.
5. If they differ, return `false`.
6. Otherwise move both pointers inward.

If all comparisons succeed, return `true`.

---


Move inward.

Skip spaces and punctuation:

```text

right -> 'm'
```

Compare:

```text

```

Continue until pointers cross.

Result:

```text

```


## Solution

```cpp
class Solution {
public:
    bool isPalindrome(string s) {

        int left = 0;
        int right = s.length() - 1;

        while(left < right) {

            while(left < right && !isalnum(s[left]))
                left++;

            while(left < right && !isalnum(s[right]))
                right--;

            if(tolower(s[left]) != tolower(s[right]))
                return false;

            left++;
            right--;
        }

        return true;
    }
};
```

## Complexity Analysis

| Metric           | Complexity |
| ---------------- | ---------- |
| Time Complexity  | O(n)       |
| Space Complexity | O(1)       |

Each character is processed at most once.

---

## Key Observation

Instead of creating a cleaned string first, we can process the original string directly.

Whenever a pointer encounters:

```text
space
punctuation
special character
```

we simply skip it.

This saves extra memory and achieves constant space complexity.
